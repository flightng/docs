本页面旨在辅助制造商/开发者/研究人员/学生/爱好者/任何人了解如何设计/制造出一个符合 FlightNG 团队设计规范的 AT32 飞控，并可在未来持续获得 FlightNG 团队的技术支持及固件更新。

## 目的

任何飞控固件都需要恰当的硬件设计来支持，而硬件设计的好坏直接影响着飞控的性能。因此，FlightNG 团队制定了一系列硬件设计规范，以保证飞控的性能和稳定性。推出该硬件设计规范有以下四个主要目的：

1. 在保持开放的基础上，向尽可能广泛的硬件提供稳定可靠快速的技术支持，尊重制造商的选型意见
2. 降低制造商/开发者/研究人员/学生/爱好者/任何人的开发成本与制造门槛
3. 降低 FlightNG 团队对广泛硬件的维护压力
4. 尽可能避免因硬件设计不当而导致的飞控性能问题
5. 消除由认知障碍造成的错误/不当信息的谣传、讹传、误传

## 补充说明

若有不在硬件设计规范中的新硬件/方案的开发需求，可联系 FlightNG 团队，我们将根据情况进行评估。同时，未来的硬件选型规格/规范应该在 FlightNG 团队的参与下进行，以确保硬件选型规格/规范的可靠性和准确性。
